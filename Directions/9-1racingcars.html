Introduction

In this assignment, you will be creating your own racing simulator. In many racing games, such as Mario Kart and Need For Speed, you are given the opportunity to choose your own car before racing it. In the assignment, the cars will be represented as objects. You will design a program such that a user can select a car and race the car, generating theorectical lap times.

Design

The design can be broken down into two parts, the design of the cars and the interactive method.

Designing the Classes

You are responsible for designing three car classes, that will all inherit from a parent Car class. The parent class consist of important characteristics that each sub class will need to also have:

  • Speed: This is an integer value that represents how fast a car is. It ranges from 0 to 100, with 0 being very slow and 100 being very fast. A car with a high speed value will benefit from a lower lap time.
  • Weight: This is an integer value that represents how heavy a car is. It ranges from 0 to 100, with 0 being light and 100 being heavy. A car with a high weight value will move slower, while a lower weight value will move faster, hence having a lower lap time.
  • Control: This is an integer value that represents how much control a car has. Cars will not have a consistent lap time each time they complete a lap. This means they will have laps where it will take them longer than usual to complete it. The integer ranges from 0 to 100, with 0 meaning no control and 100 meaning high control. A car with high control will have less variation in lap time, and it can result in a lower lap time.
  • compute_time(): This method simulates 50 laps and generates times based on a car's speed, weight, and control.
  • convert_time():This method prints out each lap time to the terminal

The Car class has been completed for you already, and due to the complexity of the compute_time() and convert_time(): methods, you'll likely want to use a form of inheritance to reuse code when constructing the other three subclasses. The three subclasses you will have to create yourself are:

  • Bullet: This car has extreme speed, but it's great speed makes it diffucult it to control. It's weight is standard to most cars.
  • Precision: This car is a master at sharp turns and staying on course. It requires multiple modifications to have this stability, which weight it down signficantly.
  • Feather: This lightweight prototype takes off many modifications to maintain a light framework. With less modifications, like engine additions and afterburners, the speed aspect is low on this car.

Hint: In the parent constructor of Car, it takes 4 arguments. You probably want to hard set the speed, control, and weight for each car, so your child __init__(), should only take one argument, and you should manually pass the stats you want to the super().

Designing the Interactive Portion

This section you have more freedom. You should prompt the user twice, once to name their car and another time to choose their car. If a user does not successfully input either one, re-prompt them. Once their car has been set up, use the proper inherited methods to generate simulated times for them.

There are no unit tests for this. You should be able to use all the cars, and the input/output should function properly.
