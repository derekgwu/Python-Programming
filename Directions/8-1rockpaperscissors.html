Chapter 8.1 Exercise: Rock Paper Scissors

Goals

  • Classes
  • User Input and Output
  • Translating Requirements to Code

Requirements

Develop your own single player text-based rock paper scissors. While most rock paper scissors games are played by two people, your Python edition of this game will support one real person while the other person is simulated by a computer (using randomly generated values).

The game should follow this format:

  • The game starts by prompting the human to enter their name. You should create a Player() class that can store a user's name, rounds won, and what move they play in a round.
  • After the user creates a name, they will enter the game and be prompted to enter rock, paper, scissor. If the user enters an invalid input, please re-prompt them to try again.
  • The CPU player should randomly generate a move. This can be done by importing the random library and use its methods. You can find it's methods here.
  • Once both sides have made a move, the program compares the move and properly awards a winner. The proper player's score is updated and the scoreboard is displayed after every round.
  • Rounds should continue to be played until the human decides to quit. Optionally, you can enable a play again option or just use the terminal cntl+c to forcefully quit a program.
